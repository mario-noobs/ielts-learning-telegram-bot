# UXWriting

## Purpose

Make UI copy guide action, reduce confusion, and set accurate expectations.

## When to Use

Anytime labels, messages, or CTAs are added or changed.

## Instructions

- Write action-first CTAs; avoid jargon; one name per concept; no placeholders as labels.
- Error messages: say what happened, why, and what to do next; avoid blame.
- Use consistent tone; show examples inline in dev tickets.

## Heuristics

- Prefer verbs to nouns in CTAs; show (Optional) explicitly; keep sentences short.

## Examples

Change "Submit" to "Save changes"; replace vague errors with actionable guidance.
