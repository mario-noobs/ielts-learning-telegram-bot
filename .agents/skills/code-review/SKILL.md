# CodeReview

## Purpose

Ensure code quality, maintainability, and security before merge.

## When to Use

Every PR.

## Instructions

- Review for correctness, test coverage, adherence to standards, security, and maintainability.
- Respond within 1 business day.

## Heuristics

- Block on missing tests or unclear logic; prefer comments over nitpicks.

## Examples

Approve small, well-tested PR; request changes for missing error handling.
