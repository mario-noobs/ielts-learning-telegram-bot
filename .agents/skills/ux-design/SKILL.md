# UXDesign

## Purpose

Design interfaces that reduce errors and speed task completion.

## When to Use

When creating or modifying user flows, forms, search, or navigation.

## Instructions

- Map task, choose the simplest pattern; use single-column forms; label fields clearly.
- Design five states; write clear error messages with action; ensure focus and keyboard support.
- Validate with 3–5 users or proxies; iterate before dev starts.

## Heuristics

- Default > settings; clarity > cleverness; reduce choices for infrequent tasks.

## Examples

Before/after flow reducing 5 clicks to 2; form error rate drops after inline validation.
