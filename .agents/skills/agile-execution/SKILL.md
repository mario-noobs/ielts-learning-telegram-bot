# AgileExecution

## Purpose

Ship value predictably with short feedback loops and explicit quality gates.

## When to Use

During planning, execution, and release of any increment.

## Instructions

- Create a small batch with clear Definition of Ready and Done (see `CHECKLISTS.md`, including observability).
- Limit WIP; review PRs within one business day; demo increments tied to outcomes.
- Release behind flags; monitor; decide to roll forward, fix fast, or roll back.

## Heuristics

- Prefer finishing over starting; if blocked > 1 day, escalate or slice smaller.

## Examples

Weekly demo showing change in a KPI; rollout to 10%, then 50%, then 100% based on signals.
